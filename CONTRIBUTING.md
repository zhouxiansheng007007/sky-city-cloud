# 参与天空农云

感谢你愿意把一个宏大想法拆成更小、更诚实的实验。

## 最有价值的贡献

1. 一个可测量、可证伪的研究问题。
2. 一份能推翻现有假设的数据或反例。
3. 一个明确写出边界条件和失效方式的工程方案。
4. 一个附带空域、天气、地面安全与社区条件的试点场地。
5. 一项让模型更透明、网站更易用或文档更准确的改进。

## 提交 Issue

请选择最接近的模板：

- Research question：作物、光谱、水热、城市气候或模型问题
- Engineering proposal：结构、材料、系留、控制、回收或安全方案
- Pilot site：场地条件、联系人、空域、气候与地面风险

请避免只写“很酷”或“不可行”。更有帮助的格式是：

> 在条件 X 下，因为证据 Y，我预计指标 Z 会发生变化；可以用实验 W 验证。

## 提交代码

1. 从 `main` 创建分支。
2. 只处理一个清晰问题。
3. 运行 `npm run build` 与 `npm test`。
4. 在 Pull Request 中说明变更、理由、影响和验证方法。
5. UI 变更需同时检查中文、英文和窄屏布局。

## 数据与引用

- 优先使用原始论文、政府或研究机构资料。
- 写明单位、时间范围、地点、样本量和不确定性。
- 不把地表温度等同于空气温度。
- 不把瞬时太阳功率等同于可用电力或全年能量。
- 不把某一种作物、气候或年份的结果外推到全球。

## 安全

不要提交可用于真实部署的未经评审高风险操作步骤。任何浮空、系留、高压、雷电、航空或载人方案，都必须把法规和第三方安全评审作为前置条件。

参与本项目即表示同意遵守 [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)。
